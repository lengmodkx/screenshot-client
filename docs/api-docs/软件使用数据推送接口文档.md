# 软件使用数据推送接口文档

## 接口概述

本文档提供给客户端系统（如桌面管控程序）用于向服务端上报软件使用数据。

- **基础URL**: `http://{server_host}:{port}`
- **Content-Type**: `application/json`
- **认证方式**: 需携带有效 Token（通过 `/client/inspection/login` 接口获取）

---

## 调用流程

客户端调用软件使用数据推送接口的完整流程：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   客户端     │────▶│  登录接口    │────▶│  获取Token   │────▶│  上报数据    │
│   (桌面)     │     │             │     │             │     │             │
└─────────────┘     └─────────────┘     └──────┬──────┘     └─────────────┘
                                                │
                                                ▼
                                         ┌─────────────┐
                                         │  Token失效   │
                                         │  重新登录    │
                                         └─────────────┘
```

---

## 认证说明

除登录接口外，所有接口需在请求头中携带访问令牌：

```http
Authorization: Bearer {accessToken}
```

> **重要提示**：
> 1. Token 通过 `/client/inspection/login` 接口获取
> 2. Token 有过期时间，过期后需要重新登录
> 3. 所有接口都需要先登录获取 Token 后才能调用

## 接口列表

### 1. 客户端登录

获取访问令牌，用于后续接口调用。

#### 请求信息

| 项目 | 内容 |
|------|------|
| 请求方式 | POST |
| 请求路径 | `/client/inspection/login` |
| 接口说明 | 客户端登录，获取访问令牌 |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| tenantName | String | 是 | 租户名称（学校/机构名称） |
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

#### 请求示例

```json
{
  "tenantName": "XX学校",
  "username": "admin",
  "password": "admin123"
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 响应码，200 表示成功 |
| data | Object | 登录成功信息 |
| data.userId | Long | 用户ID |
| data.username | String | 用户名 |
| data.deptId | Long | 部门ID |
| data.deptName | String | 部门名称 |
| data.accessToken | String | 访问令牌（后续接口使用） |
| data.refreshToken | String | 刷新令牌 |
| data.expiresTime | String | Token 过期时间 |
| msg | String | 响应消息 |

#### 响应示例

```json
{
  "code": 200,
  "data": {
    "userId": 1,
    "username": "admin",
    "deptId": 1,
    "deptName": "XX学校",
    "accessToken": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
    "refreshToken": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
    "expiresTime": "2026-03-09T18:49:39.254"
  },
  "msg": ""
}
```

### 2. 实时上报软件使用事件

实时上报单个软件使用事件（开始或结束）。

#### 请求信息

| 项目 | 内容 |
|------|------|
| 请求方式 | POST |
| 请求路径 | `/client/software/usage/realtime` |
| 接口说明 | 实时上报软件启动或停止事件 |
| 认证要求 | 需要 Token |

#### 请求头

```http
Authorization: Bearer {accessToken}
Content-Type: application/json
```

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| deviceCode | String | 是 | 设备编码（唯一标识） |
| event | String | 是 | 事件类型：`started`-开始, `stopped`-结束 |
| session | Object | 是 | 会话详细信息（必填） |

**session 对象结构：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 会话ID（唯一标识本次使用，不能为空） |
| processName | String | 是 | 进程名（如：chrome.exe，不能为空） |
| windowTitle | String | 否 | 窗口标题 |
| exePath | String | 否 | 可执行文件完整路径 |
| startTime | Long | 是 | 开始时间戳（毫秒，不能为空） |
| endTime | Long | 否 | 结束时间戳（毫秒），stopped 事件时建议填写 |
| durationSecs | Long | 否 | 使用时长（秒） |
| deviceId | Long | 否 | 设备ID（服务端分配） |
| deptId | Long | 否 | 部门ID（学校/机构ID） |
| classId | Long | 否 | 班级ID |

#### 请求示例

**软件启动事件：**

```json
{
  "deviceCode": "PC-001",
  "event": "started",
  "session": {
    "id": "sess_20240309123045_001",
    "processName": "chrome.exe",
    "windowTitle": "Google - Chrome",
    "exePath": "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe",
    "startTime": 1709965845000
  }
}
```

**软件停止事件：**

```json
{
  "deviceCode": "PC-001",
  "event": "stopped",
  "session": {
    "id": "sess_20240309123045_001",
    "processName": "chrome.exe",
    "windowTitle": "Google - Chrome",
    "exePath": "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe",
    "startTime": 1709965845000,
    "endTime": 1709966500000,
    "durationSecs": 655
  }
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 响应码，200 表示成功 |
| data | Boolean | 处理结果，`true` 表示成功 |
| msg | String | 响应消息 |

#### 响应示例

```json
{
  "code": 200,
  "data": true,
  "msg": ""
}
```

### 3. 批量上报软件使用记录

批量上报多条软件使用记录（适用于离线后补传场景）。

#### 请求信息

| 项目 | 内容 |
|------|------|
| 请求方式 | POST |
| 请求路径 | `/client/software/usage/batch` |
| 接口说明 | 批量上报软件使用记录 |
| 认证要求 | 需要 Token |

#### 请求头

```http
Authorization: Bearer {accessToken}
Content-Type: application/json
```

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| deviceCode | String | 是 | 设备编码（唯一标识，不能为空） |
| sessions | Array | 是 | 会话记录列表（不能为空，至少包含一条记录） |

**sessions 数组元素结构：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | String | 是 | 会话ID（唯一标识本次使用） |
| processName | String | 是 | 进程名（如：chrome.exe） |
| windowTitle | String | 否 | 窗口标题 |
| exePath | String | 否 | 可执行文件完整路径 |
| startTime | Long | 是 | 开始时间戳（毫秒，不能为空） |
| endTime | Long | 否 | 结束时间戳（毫秒） |
| durationSecs | Long | 否 | 使用时长（秒） |
| deviceId | Long | 否 | 设备ID（服务端分配） |
| deptId | Long | 否 | 部门ID（学校/机构ID） |

## 错误码说明

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查请求参数是否符合要求 |
| 401 | 未授权，Token 无效或过期 | 重新调用登录接口获取 Token |
| 403 | 禁止访问 | 检查用户权限 |
| 500 | 服务器内部错误 | 联系管理员或稍后重试 |

### 常见错误场景

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| 无法获取租户信息，请重新登录 | Token 无效或缺失 | 重新登录获取 Token |
| 租户不存在：xxx | 租户名称错误 | 确认租户名称是否正确 |
| 设备编码不能为空 | deviceCode 参数缺失或为空 | 检查 deviceCode 字段 |
| 事件类型不能为空 | event 参数缺失或为空 | 检查 event 字段（仅实时上报接口） |
| 会话信息不能为空 | session 参数缺失 | 检查 session 对象 |
| 会话列表不能为空 | sessions 参数缺失或为空数组 | 检查 sessions 数组（仅批量上报接口） |
| 会话ID不能为空 | session.id 缺失或为空 | 检查 session.id 字段 |
| 进程名不能为空 | session.processName 缺失或为空 | 检查 session.processName 字段 |
| 开始时间不能为空 | session.startTime 缺失 | 检查 session.startTime 字段 |

## 代码示例（Java）

### 完整调用示例

```java
import org.springframework.http.*;
import org.springframework.web.client.RestTemplate;
import java.util.*;

public class SoftwareUsageClient {

    private static final String BASE_URL = "http://server:port";
    private static final RestTemplate restTemplate = new RestTemplate();
    private String accessToken;

    /**
     * 1. 登录获取 Token
     */
    public boolean login(String tenantName, String username, String password) {
        String loginUrl = BASE_URL + "/client/inspection/login";

        Map<String, String> loginReq = new HashMap<>();
        loginReq.put("tenantName", tenantName);
        loginReq.put("username", username);
        loginReq.put("password", password);

        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        HttpEntity<Map<String, String>> entity = new HttpEntity<>(loginReq, headers);

        ResponseEntity<Map> response = restTemplate.postForEntity(loginUrl, entity, Map.class);

        if (response.getStatusCode() == HttpStatus.OK && response.getBody() != null) {
            Map<String, Object> data = (Map<String, Object>) response.getBody().get("data");
            this.accessToken = (String) data.get("accessToken");
            return true;
        }
        return false;
    }

    /**
     * 2. 上报软件使用（实时）
     */
    public boolean reportRealtime(String deviceCode, String event, SessionInfo session) {
        String pushUrl = BASE_URL + "/client/software/usage/realtime";

        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + accessToken);
        headers.setContentType(MediaType.APPLICATION_JSON);

        Map<String, Object> requestBody = new HashMap<>();
        requestBody.put("deviceCode", deviceCode);
        requestBody.put("event", event);
        requestBody.put("session", session.toMap());

        HttpEntity<Map<String, Object>> entity = new HttpEntity<>(requestBody, headers);

        try {
            ResponseEntity<Map> response = restTemplate.postForEntity(pushUrl, entity, Map.class);
            return response.getStatusCode() == HttpStatus.OK;
        } catch (Exception e) {
            // Token 可能过期，需要重新登录
            if (e.getMessage().contains("401")) {
                System.out.println("Token 过期，需要重新登录");
            }
            return false;
        }
    }

    /**
     * 3. 上报软件使用（批量）
     */
    public boolean reportBatch(String deviceCode, List<SessionInfo> sessions) {
        String pushUrl = BASE_URL + "/client/software/usage/batch";

        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + accessToken);
        headers.setContentType(MediaType.APPLICATION_JSON);

        List<Map<String, Object>> sessionList = new ArrayList<>();
        for (SessionInfo session : sessions) {
            sessionList.add(session.toMap());
        }

        Map<String, Object> requestBody = new HashMap<>();
        requestBody.put("deviceCode", deviceCode);
        requestBody.put("sessions", sessionList);

        HttpEntity<Map<String, Object>> entity = new HttpEntity<>(requestBody, headers);

        try {
            ResponseEntity<Map> response = restTemplate.postForEntity(pushUrl, entity, Map.class);
            return response.getStatusCode() == HttpStatus.OK;
        } catch (Exception e) {
            return false;
        }
    }
}

/**
 * 会话信息类
 */
class SessionInfo {
    private String id;
    private String processName;
    private String windowTitle;
    private String exePath;
    private Long startTime;
    private Long endTime;
    private Long durationSecs;

    public Map<String, Object> toMap() {
        Map<String, Object> map = new HashMap<>();
        map.put("id", id);
        map.put("processName", processName);
        map.put("windowTitle", windowTitle);
        map.put("exePath", exePath);
        map.put("startTime", startTime);
        map.put("endTime", endTime);
        map.put("durationSecs", durationSecs);
        return map;
    }

    // getters and setters...
}
```

## 注意事项

### 1. 时间戳格式
- 所有时间戳均为**毫秒级** Unix 时间戳（Long 类型）
- 示例：`1709965845000` 表示 2024-03-09 12:30:45

### 2. 会话ID唯一性
- 每次软件使用应有唯一的会话ID
- **重要**：启动（started）和停止（stopped）事件的会话ID必须保持一致
- 建议使用 UUID 或 时间戳+随机数 生成会话ID

### 3. 批量上报限制
- 单次批量上报建议不超过 **100 条**记录
- 超过限制建议分多次调用

### 4. 网络异常处理
- 建议客户端本地缓存上报失败的数据
- 网络恢复后使用批量接口补传
- 实现重试机制，避免数据丢失

### 5. Token 管理
- Token 有过期时间，过期后需要重新登录
- 建议客户端缓存 Token，在过期前或收到 401 错误时重新获取
- 不要在客户端硬编码用户名密码

### 6. 设备编码
- 设备编码是设备的唯一标识
- 需要先在服务端注册设备或通过设备注册接口注册

### 7. 租户信息
- 登录时需要提供正确的租户名称
- 所有数据都与租户关联，实现多租户隔离
