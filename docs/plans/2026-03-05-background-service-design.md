## 概述

将当前的智能黑板客户端改造为后台服务模式，实现：
1. 登录注册完成后自动最小化到系统托盘
2. 支持 Windows 开机自启
3. 自动检测并切换摄像头/屏幕截图模式
4. 全程静默运行，不打扰用户

## 组件设计

### 1. 系统托盘模块

**职责**: 提供后台运行的入口和状态反馈

**功能**:
- 创建 Windows 系统托盘图标
- 右键菜单项:
  - 显示主窗口 (Show Window)
  - 当前模式: 🎥 摄像头 / 🖥️ 屏幕截图
  - 开机自启: ✓ 开启 / ✗ 关闭
  - 查看日志 (View Logs)
  - 退出 (Exit)

**技术实现**:
```rust
// Tauri 2.0 内置 system-tray 支持
tauri::Builder::default()
    .system_tray(SystemTray::new())
    .on_system_tray_event(|app, event| {
        match event {
            SystemTrayEvent::LeftClick { .. } => show_window(app),
            SystemTrayEvent::MenuItemClick { id, .. } => handle_menu(id),
        }
    })
```

### 2. 开机自启模块

**职责**: 管理 Windows 开机启动项

**功能**:
- 安装时自动添加到启动项
- 托盘菜单可切换自启状态
- 开机启动时传递 `--autostart` 参数

**技术实现**:
使用 `tauri-plugin-autostart`:
```rust
use tauri_plugin_autostart::MacosLauncher;

// 启用开机自启
app.enable();
// 禁用开机自启
app.disable();
// 检查状态
let enabled = app.is_enabled()?;
```

**注册表位置**:
```
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run\ScreenshotClient
```

### 3. 自动模式检测模块

**职责**: 自动选择最优的视频采集模式

**检测流程**:
```
启动 ──► 检测摄像头设备 ──► 尝试初始化 ──► 成功? ──► 是 ──► 摄像头模式
                               │              │
                               │              否
                               ▼              ▼
                         失败/无权限    屏幕截图模式
```

**技术实现**:
```typescript
// 前端检测
async function detectCamera(): Promise<boolean> {
  try {
    const devices = await navigator.mediaDevices.enumerateDevices();
    const videoDevices = devices.filter(d => d.kind === 'videoinput');
    if (videoDevices.length === 0) return false;

    // 尝试获取权限
    const stream = await navigator.mediaDevices.getUserMedia({ video: true });
    stream.getTracks().forEach(t => t.stop()); // 立即释放
    return true;
  } catch {
    return false;
  }
}
```

### 4. 窗口管理模块

**职责**: 控制窗口显示/隐藏行为

**行为定义**:
| 事件 | 行为 |
|------|------|
| 应用启动（非开机自启） | 正常显示窗口，登录完成后最小化到托盘 |
| 应用启动（开机自启） | 直接最小化到托盘，不显示窗口 |
| 用户点击关闭按钮 | 最小化到托盘，不退出 |
| 用户点击托盘图标 | 显示/隐藏窗口 |
| 用户选择托盘"退出" | 完全退出应用 |

**全局快捷键**:
- `Ctrl + Shift + S`: 显示/隐藏窗口

## 错误处理策略

### 1. 静默失败原则

| 错误场景 | 处理方式 | 用户感知 |
|---------|---------|---------|
| 视频推送失败 | 记录日志，10秒后重试 | 无 |
| 网络断开 | 持续重试，指数退避 | 托盘图标变灰 |
| Token 过期 | 自动重新登录 | 无 |
| 摄像头断开 | 切换到屏幕截图模式 | 托盘菜单显示新模式 |
| 程序崩溃 | 记录日志，可选自动重启 | 无 |

### 2. 日志管理

```rust
// 日志级别
error!   // 错误，需要关注
warn!    // 警告，可以恢复
info!    // 关键流程节点

  ├── app-2026-03-04.log    # 历史日志（保留7天）
  └── app-2026-03-03.log
```

## 配置项更新

新增/修改的配置项:

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `autostart_enabled` | bool | true | 开机自启开关 |
| `capture_mode_auto` | bool | true | 自动选择采集模式 |
| `capture_mode` | string | "camera" | 当前/强制采集模式 |
| `show_window_on_start` | bool | false | 启动时是否显示窗口 |
| `tray_icon_visible` | bool | true | 托盘图标是否可见 |

## 验收标准

1. **后台运行**: 注册完成后自动最小化到托盘，不打扰用户
2. **开机自启**: 安装后自动添加到启动项，开机后自动运行
3. **自动模式**: 优先使用摄像头，无摄像头时自动切换屏幕截图
4. **托盘功能**: 右键菜单可显示窗口、查看日志、退出程序
5. **静默运行**: 除首次权限申请外，不显示任何弹窗
6. **快捷键**: Ctrl+Shift+S 可快速显示/隐藏窗口
