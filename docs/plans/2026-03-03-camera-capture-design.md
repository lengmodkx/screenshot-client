# 截图客户端 - 摄像头捕获功能设计

**日期**: 2026-03-03

## 需求概述

将截图功能从屏幕截图改为调用设备摄像头实时捕获画面，并实现软件启动时自动开始截图，无需用户手动操作。

## 确认的需求

| 需求项 | 选择 |
|--------|------|
| 截图方式 | 摄像头捕获（getUserMedia API） |
| 启动行为 | 打开软件自动开始，始终自动开始 |
| 用户交互 | 无法停止，持续工作 |

---

## 架构设计

### 技术方案

使用 **Web 方案**：通过浏览器原生 `getUserMedia` API 在前端获取摄像头流，用 `<canvas>` 捕获帧，转换为 base64 后传给 Rust 后端处理。

## 数据流设计

### 截图流程

1. **初始化** (`App.tsx` `useEffect`)
   - 调用 `loadConfig()` 加载配置
   - 调用 `initCamera()` 初始化摄像头

2. **摄像头初始化** (`initCamera`)
   - 调用 `navigator.mediaDevices.getUserMedia({ video: true })`
   - 获取视频流后设置到 `<video>` 元素
   - 创建隐藏的 `<canvas>` 用于帧捕获
   - 启动定时器，每隔 `config.interval` 秒执行一次捕获

3. **帧捕获** (`captureFrame`)
   - `canvas.drawImage(video, 0, 0)`
   - `canvas.toDataURL('image/png')` 转为 base64
   - 调用 `saveLocally(imageData)` 或 `upload`

4. **保存/上传**
   - 与现有逻辑一致，复用 `save_screenshot_to_local` 和 `upload_screenshot`

## 错误处理策略

### 摄像头相关

| 场景 | 处理方式 |
|------|----------|
| 没有摄像头设备 | 显示错误提示 "未检测到摄像头设备" |
| 用户拒绝摄像头权限 | 显示错误提示 "需要摄像头权限才能截图" |
| 摄像头被其他程序占用 | 定时重试，每 10 秒尝试重新获取流 |

### 网络相关

| 场景 | 处理方式 |
|------|----------|
| 上传失败 | 自动降级到本地保存 |
