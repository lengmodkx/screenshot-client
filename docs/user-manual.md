# 智能黑板截图客户端 - 用户操作手册

> 版本：v1.4.2
> 适用对象：教师 / 设备管理员 / 运维人员
> 平台：Windows 10 / Windows 11

---

## 一、产品简介

**智能黑板截图客户端**是部署在学校智慧黑板或多媒体教学设备上的辅助程序。它在后台自动完成以下工作：

- 📹 **实时画面推送**：将教室画面推送到管理平台，管理员可远程查看
- 📷 **定时截图**：每 5 分钟上传一张静态画面，供后台巡视使用
- 💻 **软件使用监控**：记录设备上开启的教学软件（如 PPT、希沃等）
- 🔄 **心跳保活**：保持设备在线状态
- 🖥️ **后台运行**：最小化到系统托盘，不影响正常教学

**日常使用中您几乎感知不到它的存在**，它只是安静地在后台运行。

## 二、初次使用

### 2.1 启动客户端

启动方式有三种：

**方式 1：桌面快捷方式**
双击桌面上的「截图客户端」图标。

**方式 2：开始菜单**
点击开始菜单 → 找到「ScreenshotClient」或「截图客户端」→ 点击启动。

**方式 3：开机自启动**
如果已配置开机自启，开机后会自动启动并最小化到系统托盘（右下角任务栏）。

---

### 2.2 登录

首次启动会进入登录页面。请按提示填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| **API 地址** | 管理平台的服务地址 | `http://172.16.10.11:48080` |
| **租户名称** | 平台分配的区域/学校标识 | `neimenggu` |
| **账号** | 您的登录账号 | `xcyxadmin` |
| **密码** | 您的登录密码 | `123456` |

填写完毕后点击「**登录**」按钮。

> ⚠️ 如不知道以上信息，请联系学校设备管理员或平台管理员。

---

### 2.3 设备注册（仅首次）

首次登录成功后，会自动进入设备注册页面。请如实填写：

| 字段 | 说明 | 示例 |
|------|------|------|
| **设备类型** | 选择设备类型 | 智能黑板 / 电子大屏 |
| **所属班级** | 选择本设备所在班级 | 三年一班 |
| **设备名称** | 给设备起一个易识别的名字 | 三年一班智慧黑板 |

填写完毕后点击「**注册**」。

> 💡 注册只需一次，注册信息会保存到本地，下次启动自动恢复。

注册成功后会进入主仪表盘页面。

## 四、常见操作

### 4.1 最小化到托盘

**操作**：点击窗口右上角「最小化」按钮（—）。

**效果**：窗口隐藏到系统托盘（任务栏右下角的小图标区域），后台继续工作。

> 💡 托盘图标是一个相机图标（📷）。双击托盘图标可重新显示主窗口。

---

### 4.2 从托盘恢复窗口

**操作 1**：右键点击系统托盘中的客户端图标 → 选择「显示主窗口」。

**操作 2**：左键双击托盘中的客户端图标。

---

### 4.3 查看详细日志

如需排查问题，可在设置面板中查看运行日志：

1. 点击顶部「设置」按钮
2. 切换到「日志」选项卡
3. 查看最近的运行日志（包含网络请求、错误信息等）

---

### 4.4 切换采集模式

如有需要切换为屏幕截图模式（无摄像头场景）：

1. 关闭客户端（参见 4.5）
2. 打开文件资源管理器，地址栏输入：
   ```
   %APPDATA%\ScreenshotClient\config.json
   ```
3. 用记事本打开 `config.json`
4. 找到 `"capture_mode": "camera"` 这一行
5. 改为 `"capture_mode": "screen"`
6. 保存文件
7. 重新启动客户端

---

### 4.5 退出程序

**方式 1**：点击窗口右上角「关闭」按钮（×）。

**方式 2**：点击顶部「退出」按钮。

**方式 3**：右键托盘图标 → 选择「退出」。

> ⚠️ **重要提示**：退出程序后，**视频流推送、截图上传、软件监控将全部停止**，设备在平台上会显示为离线。请在课余时间或征得管理员同意后再退出。

---

### 4.6 重新登录

如需切换账号：

1. 退出当前客户端
2. 重新启动客户端
3. 在登录页面填写新账号

## 六、设置面板说明

点击顶部「设置」按钮或托盘菜单中的「设置」打开设置面板：

### 6.1 基本设置

| 选项 | 说明 |

| 推送帧率 | 默认 2 帧/秒（一般无需修改） |

### 6.3 高级设置（需管理员权限）

| 选项 | 说明 |
|------|------|
| 心跳间隔 | 默认 30 秒 |
| 截图间隔 | 默认 5 分钟 |

### 7.1 连接状态指示

| 状态 | 含义 | 建议操作 |
|------|------|----------|
| 🟢 已连接 | 设备正常在线，所有功能正常 | 无需操作 |
| 🟡 连接中 | 正在尝试连接服务器 | 等待几秒 |
| 🔴 离线 | 无法连接服务器 | 检查网络，参见 7.2 |

### 7.2 常见状态提示信息

主界面下方状态栏会显示提示信息：

| 提示信息 | 含义 | 处理方法 |
|----------|------|----------|
| `推送中 (2 fps)` | 视频流推送正常 | 无需操作 |
| `心跳正常` | 心跳发送正常 | 无需操作 |
| `截图已上传: 14:30:25` | 截图上传成功 | 无需操作 |

## 八、常见问题

### Q1：窗口不见了，但任务栏有图标，怎么恢复？

**解决方法**：

1. 查看系统托盘（任务栏右下角）是否有相机图标（📷）
2. 双击托盘图标，主窗口会重新显示
3. 如果找不到托盘图标，点击任务栏的「^」（显示隐藏的图标）按钮

---

### Q2：登录时提示「租户名称错误」或「账号或密码错误」

**解决方法**：

1. 确认「租户名称」「账号」「密码」输入无误（注意大小写）
2. 联系平台管理员重置密码
3. 确认 API 地址正确（点击登录页面的「API 地址」字段查看）

---

### Q3：登录时提示「网络异常」

**解决方法**：

1. 检查电脑是否能上网（打开浏览器试试）
2. 检查网线/Wi-Fi 是否连接
3. 确认 API 地址正确（最常见的格式：`http://服务器IP:端口号`）
4. 等待几秒后重试（可能服务器繁忙）
5. 联系管理员确认服务器状态

---

### Q4：登录成功但显示「设备未注册」

**解决方法**：

1. 检查设备是否已注册：在设备信息卡片查看
2. 如果显示未注册或「注册信息丢失」，请联系管理员重新注册
3. 也可自行重新注册：退出登录 → 重新登录 → 完成注册流程

---

### Q5：摄像头画面是黑屏或模糊

**解决方法**：

1. 检查摄像头是否被遮挡或被其他程序占用（如视频会议软件）
2. 退出占用摄像头的程序
3. 重启客户端
4. 如仍异常，切换为屏幕截图模式（参见 4.4）

---

### Q6：右下角弹出「截图上传失败」等红色提示

**解决方法**：

1. 等待几分钟再试（可能是网络抖动）
2. 检查网络连接
3. 如果持续失败，查看设置面板 → 日志，联系管理员

---

### Q7：如何完全关闭客户端？

请使用以下任一方式：

1. 主窗口 → 点击「退出」按钮
2. 托盘菜单 → 右键 → 选择「退出」
3. 任务管理器 → 找到 `screenshot-client.exe` → 结束任务

> ⚠️ 不要用任务管理器强制结束客户端（除非卡死），因为可能丢失本地缓存数据。

---

### Q8：卸载客户端会丢失数据吗？

卸载客户端会删除：

- 客户端程序本身
- 配置文件（%APPDATA%\ScreenshotClient）
- 本地截图缓存
- 软件监控的本地数据库

**已上传到平台的数据不会丢失**（截图、软件监控记录都保存在服务器）。

如需重装后保留配置，请先备份：

```
%APPDATA%\ScreenshotClient\
```

---

### Q9：客户端占用的网络带宽大吗？

不同场景的带宽占用估算：

| 场景 | 帧率 | 单帧大小 | 上行带宽 |
|------|------|----------|----------|
| 720p 视频推送 | 2 fps | 约 100KB | 约 200 KB/s = 1.6 Mbps |
| 480p 视频推送 | 2 fps | 约 50KB | 约 100 KB/s = 0.8 Mbps |
| 截图上传 | 1 张/5分钟 | 约 100KB | 极小（可忽略） |
| 心跳保活 | 1 次/30秒 | < 1KB | 极小（可忽略） |

> 💡 单台设备占用约 1.6 Mbps 上行带宽，100 台教室设备同时在线需要约 160 Mbps 上行带宽，请规划好网络。

---

### Q10：教室断网后客户端会怎样？

客户端具备良好的**离线耐受性**：

- 视频流推送会自动暂停（断网时无法推送）
- 心跳会失败但不影响本地运行
- 截图会缓存到本地（最多缓存 15 天）
- 软件监控数据会存储到本地 SQLite 数据库
- **网络恢复后会自动重连**，断网期间的数据会自动补传

> 这意味着即使教室断网数小时甚至数天，恢复网络后所有数据都会自动续传，无需人工干预。

## 九、隐私与安全说明

### 9.1 数据采集范围

客户端**仅采集**以下信息：

| 类型 | 内容 |
|------|------|
| 视频画面 | 摄像头画面或屏幕画面（实时推流） |
| 设备信息 | 设备编码、设备名称、设备类型、所属班级 |
| 网络信息 | IP 地址、MAC 地址 |
| 软件使用 | 设备上运行的软件进程名和窗口标题 |

### 9.2 数据用途

所有数据**仅用于**：

- 教学管理和巡视
- 设备运行状态监控
- 软件使用情况统计

数据**不会**用于：

- 个人行为分析
- 商业用途
- 第三方共享

### 9.3 数据安全

- 所有通信使用 HTTPS（生产环境）
- 账号密码本地加密存储
- 设备注册后需 Token 鉴权

如对数据采集有任何疑问，请联系学校信息化负责人。
